1 — AgentCert Overview

A framework for evaluating and certifying IT-Ops agents in simulated fault-injection environments

Today's focus: Phase 1 — the fault_analyzer (Fault Bucketing) and metrics_extractor (Metrics Extraction) modules.

Key Design Principles

🔢 LLM Never Does Math

All numeric computation (sums, averages, ratios, TTD/TTR) is done deterministically in Python code. LLM is only used for text interpretation and classification.

⚠️ Worst-Case Propagation

One failure flags the entire category — safety-first approach for certification decisions.

🔁 Statistical Rigour

~30 runs per fault type ensure statistical significance. Metrics are aggregated across multiple experiment runs.

📦 Batch Processing

Traces are processed in batches (default 10–15 spans) to avoid LLM token limits while preserving context.

🎯 Ground-Truth Integration

Fault configs carry ideal trajectories and expected outcomes. Deterministic fields always override LLM-extracted values.

🛡️ Graceful Fallbacks

Every LLM call has a fallback path — classification defaults, default metrics — so the pipeline never crashes on LLM failures.

Splits a multi-fault Langfuse trace (interleaved events from multiple simultaneous faults) into per-fault buckets, each containing the complete lifecycle of one fault:

Detection → Investigation → Remediation → Verification

Class: FaultEventClassifier

LLM-based event classifier that sends batches of trace events to Azure OpenAI for classification into per-fault buckets.

Key Methods

MethodWhat it does

LLM Interaction Pattern

result, usage = await client.with_structured_output(
    model_name="extraction_model",
    messages=[{
        "role": "user",
        "content": user_message
    }],
    output_format=BatchClassificationResult,
    temperature=0.1,
    max_tokens=4000,

Note: Uses structured output (Pydantic model) so the LLM response is parsed directly into BatchClassificationResult. The user message includes detection_signals per known fault for richer context.

Fault Analyzer — Deduplication & Bucketing Logic

Deterministic Bucket Creation

Buckets are created from spans whose name starts with fault: (e.g., fault: pod-delete or fault:disk-fill). The fault name is extracted and stripped. No LLM is used for bucket creation.

Deduplication Rules

  • Active bucket exists with same fault name → event is added to existing bucket (no new bucket)
  • All previous buckets closed with same name → new bucket created with numeric suffix (e.g., pod-delete_2)

Single-Fault Fallback

If no fault: * spans are found, all events go into a single bucket named single_fault.

LLM Classification (Pass 2)

Non-fault events are batched and sent to the LLM classifier which determines:

  • Which fault bucket(s) each event relates to
  • Whether an event represents a new fault detection
  • Whether an event indicates fault mitigation

Ground-Truth from Metadata

Ground truth is extracted directly from the fault: span's metadata:

# _extract_ground_truth_from_metadata(event)
# Looks for "ground_truth" key in:
#   1. Top-level metadata dict
#   2. metadata → attributes sub-dict

metadata = {"ground_truth": {...}}
# or
metadata = {"attributes": {"ground_truth": {...}}}

Metrics Extractor — Script: span_aggregator.py

All mathematical operations are performed here — the LLM is never trusted with math.

Class: QuantitativeAggregator

MethodDescription
aggregate(partial_metrics, total_spans, span_times, bucket_metadata, validated_bucket_timestamps)Master aggregation: bucket metadata fields → timestamp resolution (validated bucket → LLM span → fallback) → first-non-null text → summable numerics → trajectory steps → boolean OR → tool call merge → ratio computation → TTD/TTR
extract_from_bucket_metadata(bucket_metadata)Extracts deterministic fields (agent info, experiment ID, injection timestamp, fault name, detected_at, mitigated_at) directly from bucket metadata
find_events_by_timestamp(timestamp, events, time_field)Static method: returns all events whose specified time field matches the given timestamp string
_parse_timestamp(ts)Parses ISO timestamps to timezone-naive UTC datetimes

Aggregation Types

  • Summable: input_tokens, output_tokens, PII instances, malicious prompts
  • First-non-null: fault name, timestamps, experiment ID, injected_fault_name, injected_fault_category, detected_fault_type, fault_target_service, fault_namespace
  • Best-match text: fault_detected (longest non-trivial description)
  • Boolean OR: pii_detection
  • List merge: tool_calls
  • Ratio: tool_selection_accuracy = correct / total
  • Computed: TTD = |detection - injection|, TTR = |mitigation - injection|
  • Timestamp resolution: LLM-validated bucket ts → LLM-identified span ts → batch fallback

Class: QualitativeAggregator

MethodDescription
aggregate(partial_observations)Averages numeric scores, computes hallucination ratio from raw counts

Aggregation Logic

  • Average: reasoning_quality_score
  • Ratio: hallucination_score = hallucination_count / total_response_count
Key Pattern: Both aggregators return a Dict[str, Any]. The extractor then does: 
for field_name, value in code_aggregated.items():
    if hasattr(llm_result, field_name) and value is not None:
        setattr(llm_result, field_name, value)
This ensures code-computed values always override LLM values.
